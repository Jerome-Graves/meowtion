\section{Evaluation}
\label{sec:evaluation}

This section reports the system's measured performance: the dataset it was trained and tested on, the
classification accuracy of each cascade stage, the benefit of the audio confirmation stage, the
on-device resource footprint and inference latency, power draw and battery life, and the time taken to
deliver a model over the air. All on-device figures are measured on the collar hardware
(\cref{sec:hardware}); accuracy figures are reported on a held-out test split of the labelled clips.

\subsection{Benefit of the audio confirmation stage}

Eating and drinking are nearly identical to an accelerometer but distinct by sound, so they are the
case the audio stage exists to resolve. Adding the gated audio stage raises eat/drink accuracy from
\ph{IMU-only eat/drink accuracy \%} to \ph{cascade eat/drink accuracy \%}.

dominant RAM cost of inference; the audio window buffer and the BLE stack make up most of the rest.

\subsection{Limitations}

The current evaluation is single-cat and within-subject; generalisation across cats and households is
addressed as future work (\cref{sec:future-work}). The figures above are for the behaviour set used in
testing; accuracy on a larger or more fine-grained label set should be re-measured as the set grows.
